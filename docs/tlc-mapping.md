# TLC to Jinja2 Mapping

bakeC replaces MATLAB's Target Language Compiler (TLC) with Jinja2
templates while preserving the same architectural separation between
model data and output syntax.

## Concept Mapping

| MATLAB / TLC                  | bakeC / Jinja2                        |
|-------------------------------|---------------------------------------|
| `.rtw` file (flat text IR)    | Python dict from YAML parser          |
| `.tlc` template file          | `.j2` Jinja2 template                 |
| `%<variable>`                 | `{{ variable }}`                      |
| `%foreach idx = N`            | `{% for idx in range(N) %}`           |
| `%if condition`               | `{% if condition %}`                  |
| `%function`                   | `{% macro name() %}`                  |
| `%include "file.tlc"`         | `{% include "file.j2" %}`             |
| `%selectfile`                 | Separate template per output file     |
| System target file (`.stf`)   | Platform YAML (`platforms/*.yaml`)    |
| `rtwtypes.h`                  | `_types.h` (generated from template)  |
| Block TLC (`block.tlc`)       | `templates/blocks/*.c.j2`             |
| `LibBlockParameterValue`      | `{{ block.params.coefficients }}`     |

## Side-by-Side Examples

### Example 1: Loop unrolling decision

**TLC:**
```
%if N <= 8
  %foreach i = N
    result += coeffs[%<i>] * phi(%<i>);
  %endforeach
%else
  for (i = 0; i < %<N>; i++) {
    result += coeffs[i] * phi(i);
  }
%endif
```

**Jinja2:**
```jinja2
{% if block.unroll %}
{% for i in range(block.params.num_basis_functions) %}
    result += coeffs[{{ i }}] * phi({{ i }});
{% endfor %}
{% else %}
    for (i = 0; i < {{ block.params.num_basis_functions }}; i++) {
        result += coeffs[i] * phi(i);
    }
{% endif %}
```

### Example 2: Type-portable literals

**TLC:**
```
%<LibRealNonFinite(0.0)>
real_T threshold = 1.0e-10%<TypeSuffix>;
```

**Jinja2:**
```jinja2
real_T threshold = 1.0e-10{{ type_suffix }};
```

The `type_suffix` variable resolves to `"f"` for `float` platforms
(Cortex-M4) and `""` for `double` platforms (desktop).

### Example 3: Traceability banner

**TLC:**
```
/* Generated by Embedded Coder %<Version>
 * Model: %<ModelName>
 * Date:  %<TLC_TIME>
 */
```

**Jinja2:**
```jinja2
/* Generated by bakeC v{{ meta.version }}
 * Model:    {{ meta.model_path }} [sha256:{{ meta.model_hash }}]
 * Platform: {{ meta.platform_path }} [sha256:{{ meta.platform_hash }}]
 * Date:     {{ meta.timestamp }}
 *
 * DO NOT EDIT -- generated file.
 */
```

## Key Differences

1. **Data binding** — TLC reads from a flat `.rtw` text file with
   `%<>` references. Jinja2 receives a structured Python dict, enabling
   nested access (`block.params.knots`) and list comprehensions.

2. **Output routing** — TLC uses `%selectfile` to direct output to
   different files within a single template. bakeC uses one template per
   output file, with the engine calling `render()` for each.

3. **Extensibility** — Adding a new block type in TLC requires a new
   `.tlc` file registered in the system target file. In bakeC, add a
   `.j2` fragment in `templates/blocks/` and reference it from the main
   template via `{% include %}`.
